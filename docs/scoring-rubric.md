# Scoring Rubric

This rubric is not a benchmark.

It is a practical, document-based design review rubric for comparing AI coding-agent skills.

## Criteria

Each criterion can be scored from 1 to 10:

1. **Generality**: Is it applicable to various projects?
2. **Clarity**: Are the instructions clear and easy to understand?
3. **Executability**: Is it ready for immediate use in real tasks?
4. **Over-edit prevention**: Does it reduce unnecessary changes and refactoring?
5. **Verification discipline**: Does it enforce verification and evidence?
6. **Uncertainty handling**: Does it separate reasoning from verification and manage uncertainty?
7. **Complex-task fit**: Is it strong for complex modifications, debugging, and multi-file changes?
8. **Lightweight for small tasks**: Is it lightweight enough for trivial tasks?
9. **Adoption ease**: Is it easy to install and start using?
10. **Originality**: Does it have unique and differentiated approaches?

## Notes

Scores should not be treated as absolute performance measurements.

A real benchmark would require the same coding tasks, same model, same environment, and measured outcomes such as:

- success rate
- unnecessary changed lines
- failed verification
- time to completion
- regression rate
- number of required user interventions
