## 📊 벤치마크 비교 결과

ChatGPT와 Gemini 두 관점에서 **Safe Code Agent**, **Karpathy Skills**, **Grill Me** 세 도구를 10개 항목으로 평가한 결과입니다.

| 관점 | Safe Code Agent | Karpathy Skills | Grill Me |
|---|:---:|:---:|:---:|
| ChatGPT 기준 | 8.00 | **8.45** | 6.75 |
| Gemini 기준 | 8.0 | **8.1** | 7.4 |

### 평가 항목 (10개)

| # | 항목 | 설명 |
|---|---|---|
| 1 | 범용성 | 다양한 프로젝트에 적용 가능한가? |
| 2 | 명확성 | 지침이 명확하고 이해하기 쉬운가? |
| 3 | 실행 가능성 | 실제 작업에 즉시 투입 가능한가? |
| 4 | 과수정 방지 | 불필요한 수정과 리팩토링을 줄이는가? |
| 5 | 검증 강제력 | 검증을 강제하고 증거를 남기는가? |
| 6 | 불확실성 관리 | 추론과 검증을 분리해서 불확실성을 관리하는가? |
| 7 | 복잡한 작업 대응 | 복잡한 수정, 디버깅, 다파일 변경에 강한가? |
| 8 | 작은 작업 가벼움 | 사소한 작업도 가볍게 쓸 수 있는가? |
| 9 | 도입 쉬움 | 바로 적용하기 쉬운가? |
| 10 | 독창성 | 차별화된 접근과 독자성이 있는가? |

### 도구별 포지셔닝

- 🏆 **Karpathy Skills** — 가장 좋은 기본값. 짧고 범용적이고 도입이 쉬움
- 🛡️ **Safe Code Agent** — 기본값으로는 조금 무겁지만, **복잡한 코드 수정·검증 투명성·과수정 방지**에는 더 구체적
- ❓ **Grill Me** — 코드 수정용이 아니라 설계/계획 검증용으로 보는 편이 정확함

> **핵심**: 실제 코딩 작업에서는 목적이 다르므로 평균 점수보다 항목별 강점이 더 중요함

## ✅ 최종 결론

### 실제로 도움이 된 부분

safe-code-agent는 **복잡한 코드 수정과 검증 투명성**에는 실제로 도움이 됐다.

특히 강했던 부분:

- ✅ 수정 전 코드 확인
- ✅ 최소 수정 원칙
- ✅ 실패 케이스 실제 실행
- ✅ 실패 원인 재현
- ✅ 같은 명령으로 수정 후 재검증

### 한계

- ⚠️ 설계 위반을 처음부터 완전히 막지는 못함
- ⚠️ smoke 설계 실수는 사람이 보강해야 함
- ⚠️ 긴 검증 전략은 사람이 판단해야 함

### 한 줄 요약

> **safe-code-agent는 "AI가 덜 틀리게 만드는 마법"이 아니라,  
> AI가 틀렸을 때 그 실수를 빨리 드러내고 증거 기반으로 고치게 만드는  
> 안전한 작업 방식에 가깝다.**
